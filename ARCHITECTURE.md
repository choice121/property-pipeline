# Architecture

  ## System Overview

  Property Pipeline is a two-part system:
  - A **Python backend** that handles scraping, image downloading, and data storage
  - A **React frontend** that provides the dashboard for viewing, editing, and publishing

  These two parts communicate over a local REST API (FastAPI).

  ---

  ## Tech Stack

  ### Backend
  | Technology | Purpose |
  |---|---|
  | Python 3.11+ | Primary backend language |
  | FastAPI | REST API framework |
  | HomeHarvest | Property scraping library (Zillow, Realtor.com, Redfin) |
  | SQLite | Internal database (dev — no setup needed, single file) |
  | SQLAlchemy | ORM for database interactions |
  | httpx | Async HTTP client for downloading images |
  | Pillow | Image processing and validation |
  | python-dotenv | Environment variable management |
  | uvicorn | ASGI server to run FastAPI |

  ### Frontend
  | Technology | Purpose |
  |---|---|
  | React 18 | UI framework |
  | Vite | Build tool and dev server |
  | React Router v6 | Page routing |
  | TanStack Query (React Query) | Server state, data fetching, caching |
  | Axios | HTTP client for API calls |
  | Tailwind CSS | Utility-first styling |
  | shadcn/ui | Accessible, unstyled UI component library |

  ### Storage
  | What | Where |
  |---|---|
  | Property data | SQLite database file (backend/data/pipeline.db) |
  | Downloaded images | Local folder (backend/storage/images/{property_id}/) |
  | Configuration | .env file (never committed to git) |

  ## Database Schema (Internal — NOT Choice Properties)

  This is the schema for this tool's own SQLite database. It has nothing to do with the live website.

  ```sql
  CREATE TABLE properties (
    -- Identity
    id                   TEXT PRIMARY KEY,      -- e.g. PP-A1B2C3 (generated by this tool)
    source               TEXT NOT NULL,         -- 'zillow' | 'realtor' | 'redfin'
    source_url           TEXT,                  -- original listing URL
    source_listing_id    TEXT,                  -- ID from the source platform

    -- Status
    status               TEXT DEFAULT 'scraped',
    -- Values: scraped | edited | ready | published

    -- Core listing fields (editable)
    title                TEXT,
    address              TEXT,
    city                 TEXT,
    state                TEXT,
    zip                  TEXT,
    county               TEXT,
    lat                  REAL,
    lng                  REAL,
    bedrooms             INTEGER,
    bathrooms            REAL,
    half_bathrooms       INTEGER,
    square_footage       INTEGER,
    lot_size_sqft        INTEGER,
    monthly_rent         INTEGER,
    property_type        TEXT,
    year_built           INTEGER,
    description          TEXT,
    available_date       TEXT,
    parking              TEXT,
    pets_allowed         BOOLEAN,
    pet_details          TEXT,
    smoking_allowed      BOOLEAN,
    lease_terms          TEXT,                  -- JSON array
    amenities            TEXT,                  -- JSON array
    appliances           TEXT,                  -- JSON array
    utilities_included   TEXT,                  -- JSON array
    flooring             TEXT,                  -- JSON array
    heating_type         TEXT,
    cooling_type         TEXT,
    laundry_type         TEXT,
    virtual_tour_url     TEXT,

    -- Images
    original_image_urls  TEXT,                  -- JSON array — raw scraped URLs, never changed
    local_image_paths    TEXT,                  -- JSON array — relative paths in /storage/images/

    -- Edit audit trail
    original_data        TEXT,                  -- Full JSON snapshot of raw scraped data, never changed
    edited_fields        TEXT DEFAULT '[]',     -- JSON array of field names the owner edited

    -- Publishing (filled in by Stage 7)
    published_at         TEXT,
    choice_property_id   TEXT,                  -- The ID assigned in Choice Properties DB after publish

    -- Timestamps
    scraped_at           TEXT DEFAULT CURRENT_TIMESTAMP,
    updated_at           TEXT DEFAULT CURRENT_TIMESTAMP
  );
  ```

  ## API Reference

  ### Scraper
  ```
  POST /api/scrape
  Body: {
    location: string,          // e.g. "Austin, TX" or "78701"
    source: string,            // "zillow" | "realtor" | "redfin"
    listing_type: string,      // "for_rent" | "for_sale"
    min_price: number | null,
    max_price: number | null,
    bedrooms: number | null,
    property_type: string | null
  }
  Response: {
    count: number,
    properties: Property[]
  }
  ```

  ### Properties
  ```
  GET    /api/properties                    # list all (query params: status, search, sort)
  GET    /api/properties/{id}              # get single property
  PUT    /api/properties/{id}              # update fields (triggers edited_fields tracking)
  DELETE /api/properties/{id}              # delete property and its images
  ```

  ### Images
  ```
  GET    /api/images/{property_id}/{filename}       # serve stored image file
  DELETE /api/properties/{id}/images/{index}       # remove one image by index
  PUT    /api/properties/{id}/images/reorder        # reorder images
  Body: { order: [0, 2, 1, 3] }
  ```

  ### Publisher (Stage 7 only)
  ```
  POST /api/publish/{id}
  Response: {
    ok: boolean,
    choice_property_id: string,
    message: string
  }
  ```

  ## How Publishing Works (Stage 7)

  1. Backend reads the property record and its local images
  2. Each image is uploaded to owner's ImageKit account → returns URL + fileId
  3. Property record is inserted into Choice Properties Supabase `properties` table
  4. Supabase status set to `active` — listing goes live immediately on the website
  5. This tool updates local record: status = `published`, published_at = now(), choice_property_id = returned ID

  **Environment variables required for Stage 7:**
  ```
  SUPABASE_URL=
  SUPABASE_SERVICE_ROLE_KEY=
  IMAGEKIT_PRIVATE_KEY=
  IMAGEKIT_PUBLIC_KEY=
  IMAGEKIT_URL_ENDPOINT=
  CHOICE_LANDLORD_ID=
  ```

  These are provided by the owner at Stage 7 time. They are never committed to git.

  ## How This Connects to Choice Properties

  | Choice Properties | This Tool |
  |---|---|
  | Supabase `properties` table | Writes one row on publish |
  | ImageKit account | Uploads images on publish |
  | `status = 'active'` | Set on insert — listing goes live immediately |
  | `landlord_id` | Uses CHOICE_LANDLORD_ID from .env |
  | `photo_urls` | Set to ImageKit URLs returned after upload |
  | `photo_file_ids` | Set to ImageKit fileIds for future deletion support |
